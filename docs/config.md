# Configuration

Полная схема `config.toml`. Каждая таблица описывает свою секцию, столбец **«Допустимые значения»** перечисляет, что rterm проглотит, и что отвергнет / зажмёт в реалистичный диапазон.

Связанные документы:

- [Keybindings](keybindings.md) — секция `[[keybindings]]` и полный список actions.
- [Themes & Appearance](themes.md) — `[appearance]` и встроенные темы.
- [Getting started](getting-started.md) — где лежит `config.toml` и как hot-reload.

## `[font]`

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `family` | string | `""` (системный моноширинный) | имя установленного семейства или пустая строка | `"JetBrains Mono"`, `"Fira Code"`, `"MesloLGS NF"`, …. Список установленных моноширинных: `rterm --list-fonts`. |
| `size` | float | `13.0` | `6.0 .. 96.0`; nan / inf отвергаются | Размер в пунктах. CLI-флаг `--font-size N` перекрывает. |
| `bold_is_bright` | bool | `true` | `true` / `false` | Поведение xterm: жирный текст в одном из 8 базовых цветов автоматически переходит в bright-вариант. Поставь `false` для тем, где жирный используется только как выделение без смены цвета. |

## `[window]`

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `width` | int | `1024` | `≥ 320` (значения ниже клампятся) | Стартовая ширина окна, **логические** пиксели. |
| `height` | int | `640` | `≥ 200` (значения ниже клампятся) | Стартовая высота окна, логические пиксели. |
| `opacity` | float | `1.0` | `0.0 .. 1.0`; nan → `1.0` | От `0.0` (прозрачно) до `1.0` (непрозрачно). Требует поддержки прозрачности у композитора — Wayland / macOS / Windows с включённым DWM. |
| `os_decorations` | bool | `false` | `true` / `false` | Когда `false` — rterm рисует свой title-bar и кнопки; `true` — отдаёт это WM (полезно на тайловых WM). |

## `[shell]`

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `program` | string | `""` (= `$SHELL` на Unix, `powershell.exe` на Windows) | абсолютный путь или пустая строка | Бинарник шелла. Пустая строка наследует `$SHELL`. |
| `args` | `[string]` | `[]` | любые CLI-аргументы | Аргументы, передаваемые шеллу. |
| `env` | `{string: string}` | `{}` | произвольная карта ключ→значение | Дополнительные переменные окружения. Применяются ПОСЛЕ встроенных `TERM` / `COLORTERM` / `TERM_PROGRAM`, поэтому пользовательский ключ перекрывает дефолт. Родительские env-переменные при этом не сбрасываются — это аддитивная карта. |

Пример:

```toml
[shell]
program = "/bin/zsh"
args    = ["-l"]

[shell.env]
LANG          = "en_US.UTF-8"
RUST_BACKTRACE = "1"
```

## `[terminal]`

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `scrollback` | int | `10000` | `0 .. usize::MAX` (рекомендуется ≤ `100_000`) | Сколько строк хранить в скроллбэке каждой панели. `0` отключает скроллбэк целиком. Память ≈ `lines × cols × 16` байт. |
| `save_scrollback_on_exit` | bool | `false` | `true` / `false` | На выходе дамп скроллбэка фокусной панели в `$XDG_CACHE_HOME/rterm/scrollback-<ts>.txt`. Эквивалент действия `save_scrollback`. |
| `restore_session` | bool | `false` | `true` / `false` | На выходе сохранять список вкладок (одна cwd на вкладку) в `$XDG_CACHE_HOME/rterm/session.toml` и восстанавливать на следующем старте. Сплиты не сохраняются. |
| `scroll_on_output` | bool | `false` | `true` / `false` | Любой новый вывод сбрасывает `scroll_offset = 0` (поведение «tail»). По умолчанию rterm не дёргает скролл, пока пользователь читает старую строку. |
| `cursor_blink` | bool | `true` | `true` / `false` | Главный переключатель мигания курсора. `false` отрисовывает курсор без мигания, что бы ни просил шелл через DECSCUSR. |
| `show_scrollbar` | bool | `true` | `true` / `false` | Тонкий индикатор скролла справа. `false` прячет его полностью. |
| `tab_silence_ms` | int (ms) | `5000` | `≥ 0`; `0` отключает | Сколько мс должно пройти после последнего вывода, чтобы для вкладки сработало edge-событие `tab.silence`. |
| `bell_visual` | bool | `true` | `true` / `false` | Визуальная вспышка экрана на BEL (`\a`) или `rterm.bell()` из плагина. |
| `bell_urgent` | bool | `true` | `true` / `false` | Дёргать оконный менеджер (urgency-хинт / dock-badge) на BEL, когда окно не в фокусе. |
| `slow_command_ms` | int (ms) | `10000` | `≥ 0`; `0` отключает | Порог для события `pane.slow_command`. Команда, помеченная OSC 133;C / 133;D (shell-integration), выполняющаяся дольше — триггерит edge-событие и, если окно не в фокусе, пинг в таскбар. |
| `allow_osc52` | bool | `false` | `true` / `false` | Разрешить шеллу записывать в системный clipboard через `ESC ] 52 ; c ; <base64> ESC \`. По умолчанию **выключено** — враждебный pager-вывод или цепочка пайпов может перезаписать буфер между вашим «копировать» и «вставить» (реальный phishing-сценарий). Включай, только если завязан на tmux / mosh / SSH clipboard forwarding. Заблокированные записи фиксируются событием `osc52.blocked` (плагин может показать тоаст). |

## `[guake]`

Guake-style «drop-down» терминал: окно по нажатию бинда «выскакивает» поверх остальных, занимая часть экрана (как в [Guake](https://github.com/Guake/guake) / [Yakuake](https://apps.kde.org/yakuake/) / `tmux popup`). Удобно для quick-access без переключения столов.

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `enabled` | bool | `false` | `true` / `false` | Мастер-выключатель. Если `false`, action `toggle_guake` — no-op. |
| `position` | string | `"top"` | `"top"` / `"bottom"` / `"full"` | К какому краю экрана прижимать «дроп». `"full"` — на весь монитор. |
| `height_pct` | int | `50` | `10..=100` | Какую долю высоты монитора занимает окно (для `"top"` / `"bottom"`). Игнорируется для `"full"`. |
| `width_pct` | int | `100` | `20..=100` | Какую долю ширины монитора занимает окно. |

### Пример: F11 → выкатывание сверху на пол-экрана

```toml
[guake]
enabled = true
position = "top"     # вылазит сверху
height_pct = 50      # на половину высоты экрана
width_pct = 100      # на всю ширину

[[keybindings]]
keys   = "F11"
action = "toggle_guake"
```

После этого `F11` (когда окно rterm в фокусе) сворачивает / разворачивает окно в дроп-режиме. Повторное нажатие — minimize.

### Платформа

- **X11 / Windows / macOS** — `set_outer_position` + `request_inner_size` работают; окно точно ложится на нужный край.
- **Wayland** — композитор владеет позиционированием; `"top"` / `"full"` падают в `set_maximized(true)`, `"bottom"` только меняет размер (где появится окно — решает композитор).

### Системный (глобальный) хоткей

Бинд `toggle_guake` срабатывает **только когда окно rterm в фокусе** (или восстанавливается из minimize через taskbar/Alt-Tab). Чтобы F11 поднимал rterm даже из-под другого приложения — это уровень WM/DE, rterm специально не тянет глобальный shortcut-крейт. Варианты:

- **GNOME**: Settings → Keyboard → Custom Shortcut → команда `wmctrl -xa rterm` (или `xdotool search --class rterm windowactivate`), клавиша `F11`.
- **KDE / KWin**: System Settings → Shortcuts → Custom Shortcuts → New Global Shortcut → Command/URL → та же команда.
- **Sway / Hyprland / i3**: добавь в конфиг WM строку вроде `bindsym F11 exec wmctrl -xa rterm` (или `hyprctl dispatch focuswindow class:rterm`).
- **Windows**: PowerToys Keyboard Manager → ремап F11 на скрипт активации окна.

WM-сторонний шорткат поднимает / фокусит окно, а уже внутри окна сам бинд `F11 → toggle_guake` срабатывает за reposition.

## `[colors]`

Все поля опциональные. Каждое — RGB-тройка байт `[r, g, b]`. Отсутствующий ключ означает «оставить встроенный xterm-дефолт» (или унаследовать из выбранной темы `[appearance].theme`).

| Ключ | Допустимые значения | Назначение |
|------|---------------------|------------|
| `fg`, `bg` | `[u8, u8, u8]` каждый компонент `0..=255` | Цвет текста и фона по умолчанию. |
| `cursor` | `[u8, u8, u8]` | Фиксированный цвет блока курсора. Если не задан — инвертированный фон ячейки (xterm-стиль). |
| `black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white` | `[u8, u8, u8]` | 8 базовых ANSI-цветов (индексы 0..7). |
| `bright_black` … `bright_white` | `[u8, u8, u8]` | 8 «ярких» цветов (индексы 8..15). |

Пример «Solarized Dark» поверх любой выбранной темы:

```toml
[colors]
fg      = [131, 148, 150]
bg      = [0,   43,  54]
cursor  = [220, 50,  47]

black   = [7,   54,  66]
red     = [220, 50,  47]
green   = [133, 153, 0]
yellow  = [181, 137, 0]
blue    = [38,  139, 210]
magenta = [211, 54,  130]
cyan    = [42,  161, 152]
white   = [238, 232, 213]

bright_black   = [0,   43,  54]
bright_red     = [203, 75,  22]
bright_green   = [88,  110, 117]
bright_yellow  = [101, 123, 131]
bright_blue    = [131, 148, 150]
bright_magenta = [108, 113, 196]
bright_cyan    = [147, 161, 161]
bright_white   = [253, 246, 227]
```

## `[appearance]`

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `theme` | string | `""` (= `default`) | один из: `default` `dark` `dracula` `solarized-dark` `solarized-light` `nord` `gruvbox-dark` `light` или пустая строка | Имя встроенной темы. Тема применяется первой, **поверх** неё накладываются `[colors]` (если заданы). При переключении через `cycle_theme` поле перезаписывается. |

Подробнее: [Themes & Appearance](themes.md).

## `[[keybindings]]`

Массив таблиц. Каждая запись — пара `keys` + `action`. Пользовательские бинды перекрывают встроенные.

| Ключ | Тип | По умолчанию | Допустимые значения | Описание |
|------|-----|--------------|---------------------|----------|
| `keys` | string | — (обязательное) | спецификация «модификаторы + клавиша» (см. [Keybindings](keybindings.md#keys-format)) | Само сочетание, например `Ctrl+Shift+T`. |
| `action` | string | — (обязательное) | имя action'a; должно быть в `rterm --list-actions` или зарегистрировано через `rterm.register_action(name, fn)` | Что делать при нажатии. |

```toml
[[keybindings]]
keys   = "Ctrl+T"
action = "new_tab"

[[keybindings]]
keys   = "F2"
action = "search"
```

Полная справка по `keys` синтаксису и список встроенных action — [Keybindings](keybindings.md).
